# FreshFace 改进计划 5 - 引导页状态持久化优化

## 一、需求描述

### 1.1 问题背景
当前 FreshFace 应用的 **Feature Highlights（功能亮点引导）** 功能存在严重 Bug：
- 用户每次切换到新的 Tab（Home/Products/Routine/Analytics/Settings）时，都会弹出引导提示页
- 即使已经查看过该 Tab 的引导，**App 重启后仍会重复显示**
- 这严重影响用户体验，不符合 iOS 应用"首次使用才显示引导"的常规设计模式

### 1.2 问题根因分析
| 文件 | 问题 | 影响 |
|------|------|------|
| `FeatureHighlights.swift` | `hasSeen*Highlight` 变量仅使用 `@Published` 声明，存储在内存中 | App 重启后状态丢失 |
| `FeatureHighlightsManager` | 单例类但内部状态未持久化 | 无法跨会话保持用户偏好 |
| `AppRootView.swift` | 每次 Tab 切换都调用 `shouldShowHighlight()` | 因状态重置为 false，始终返回 true |

### 1.3 期望行为
| 场景 | 当前行为 | 期望行为 |
|------|----------|----------|
| 首次下载并打开 App | 显示 Onboarding + 各Tab引导 | ✅ 保持不变 |
| 第二次及以后打开 App | ❌ 仍显示所有Tab引导 | ✅ 不再显示任何引导 |
| 点击 "Reset All Tips" 按钮 | 重置后再次显示 | ✅ 保持不变 |
| 卸载重装 App | - | ✅ 视为新用户，重新显示引导 |

#### ⚠️ 技术限制说明
**重要发现**：经过代码审查，原计划中使用的 `@AppStorage` 方案存在技术错误：

| 属性包装器 | 可用位置 | 说明 |
|------------|----------|------|
| `@AppStorage` | ❌ 仅限 `struct View` | SwiftUI 专用，不能在 class 中使用 |
| `@Published` + `UserDefaults` | ✅ 可用于 `class ObservableObject` | 项目现有模式 |

采用与项目现有 [LocalProductRepository](file:///Volumes/ORICO-APFS/app/20260407/freshface/freshface/freshface/Data/Repositories/LocalProductRepository.swift) 和 [LocalRoutineRepository](file:///Volumes/ORICO-APFS/app/20260407/freshface/freshface/freshface/Data/Repositories/LocalRoutineRepository.swift) 相同的模式：

```swift
// 现有代码参考 (LocalProductRepository.swift)
private var savedProducts: [Product] {
    get {
        guard let data = UserDefaults.standard.data(forKey: productsKey) else {
            return []
        }
        return (try? JSONDecoder().decode([Product].self, from: data)) ?? []
    }
    set {
        if let data = try? JSONEncoder().encode(newValue) {
            UserDefaults.standard.set(data, forKey: productsKey)
        }
    }
}
```

```swift
// ============================================
// 改进前 (当前有问题的代码)
// ============================================
class FeatureHighlightsManager: ObservableObject {
    static let shared = FeatureHighlightsManager()

    // ❌ 问题：仅内存变量，App重启即丢失
    @Published var hasSeenHomeHighlight = false
    @Published var hasSeenProductsHighlight = false
    @Published var hasSeenRoutineHighlight = false
    @Published var hasSeenAnalyticsHighlight = false
    @Published var hasSeenSettingsHighlight = false

    private init() {}
    
    // ... 其他方法
}

// ============================================
// 改进后 (目标代码 - 使用 UserDefaults 持久化)
// ============================================
class FeatureHighlightsManager: ObservableObject {
    static let shared = FeatureHighlightsManager()

    // MARK: - UserDefaults Keys
    private enum StorageKey: String {
        case home = "ff_highlight_home"
        case products = "ff_highlight_products"
        case routine = "ff_highlight_routine"
        case analytics = "ff_highlight_analytics"
        case settings = "ff_highlight_settings"
    }

    // MARK: - Published Properties (内存缓存)
    @Published private(set) var hasSeenHomeHighlight: Bool = false
    @Published private(set) var hasSeenProductsHighlight: Bool = false
    @Published private(set) var hasSeenRoutineHighlight: Bool = false
    @Published private(set) var hasSeenAnalyticsHighlight: Bool = false
    @Published private(set) var hasSeenSettingsHighlight: Bool = false

    private init() {
        loadPersistedStates()
    }

    // MARK: - Persistence Methods
    private func loadPersistedStates() {
        hasSeenHomeHighlight = UserDefaults.standard.bool(forKey: StorageKey.home.rawValue)
        hasSeenProductsHighlight = UserDefaults.standard.bool(forKey: StorageKey.products.rawValue)
        hasSeenRoutineHighlight = UserDefaults.standard.bool(forKey: StorageKey.routine.rawValue)
        hasSeenAnalyticsHighlight = UserDefaults.standard.bool(forKey: StorageKey.analytics.rawValue)
        hasSeenSettingsHighlight = UserDefaults.standard.bool(forKey: StorageKey.settings.rawValue)
    }

    private func persistState(_ value: Bool, for key: StorageKey) {
        UserDefaults.standard.set(value, forKey: key.rawValue)
    }

    // MARK: - Public API
    func shouldShowHighlight(for tab: AppCoordinator.AppTab) -> Bool {
        switch tab {
        case .home:
            return !hasSeenHomeHighlight
        case .products:
            return !hasSeenProductsHighlight
        case .routine:
            return !hasSeenRoutineHighlight
        case .analytics:
            return !hasSeenAnalyticsHighlight
        case .settings:
            return !hasSeenSettingsHighlight
        }
    }

    func markAsSeen(tab: AppCoordinator.AppTab) {
        switch tab {
        case .home:
            hasSeenHomeHighlight = true
            persistState(true, for: .home)
        case .products:
            hasSeenProductsHighlight = true
            persistState(true, for: .products)
        case .routine:
            hasSeenRoutineHighlight = true
            persistState(true, for: .routine)
        case .analytics:
            hasSeenAnalyticsHighlight = true
            persistState(true, for: .analytics)
        case .settings:
            hasSeenSettingsHighlight = true
            persistState(true, for: .settings)
        }
    }

    func resetAll() {
        let allKeys: [StorageKey] = [.home, .products, .routine, .analytics, .settings]
        
        hasSeenHomeHighlight = false
        hasSeenProductsHighlight = false
        hasSeenRoutineHighlight = false
        hasSeenAnalyticsHighlight = false
        hasSeenSettingsHighlight = false
        
        allKeys.forEach { UserDefaults.standard.removeObject(forKey: $0.rawValue) }
    }
}
```

### 2.3 检验标准

#### 单元测试用例
```swift
import XCTest
@testable import FreshFace

final class FeatureHighlightsManagerTests: XCTestCase {

    override func setUpWithError() throws {
        super.setUp()
        FeatureHighlightsManager.shared.resetAll()
    }

    override func tearDownWithError() throws {
        FeatureHighlightsManager.shared.resetAll()
        super.tearDown()
    }

    // 测试场景 1：首次使用应返回true
    func testShouldShowHighlight_FirstTime_ReturnsTrue() {
        let manager = FeatureHighlightsManager.shared
        
        XCTAssertTrue(manager.shouldShowHighlight(for: .home))
        XCTAssertTrue(manager.shouldShowHighlight(for: .products))
        XCTAssertTrue(manager.shouldShowHighlight(for: .routine))
        XCTAssertTrue(manager.shouldShowHighlight(for: .analytics))
        XCTAssertTrue(manager.shouldShowHighlight(for: .settings))
    }

    // 测试场景 2：标记已查看后应返回false
    func testShouldShowHighlight_AfterMarkingSeen_ReturnsFalse() {
        let manager = FeatureHighlightsManager.shared
        
        manager.markAsSeen(tab: .home)
        
        XCTAssertFalse(manager.shouldShowHighlight(for: .home))
        XCTAssertTrue(manager.shouldShowHighlight(for: .settings))
    }

    // 测试场景 3：标记多个Tab后逐一验证
    func testShouldShowHighlight_MultipleTabsMarked() {
        let manager = FeatureHighlightsManager.shared
        
        manager.markAsSeen(tab: .home)
        manager.markAsSeen(tab: .products)
        
        XCTAssertFalse(manager.shouldShowHighlight(for: .home))
        XCTAssertFalse(manager.shouldShowHighlight(for: .products))
        XCTAssertTrue(manager.shouldShowHighlight(for: .routine))
        XCTAssertTrue(manager.shouldShowHighlight(for: .analytics))
        XCTAssertTrue(manager.shouldShowHighlight(for: .settings))
    }

    // 测试场景 4：重置后应重新返回true
    func testShouldShowHighlight_AfterReset_ReturnsTrue() {
        let manager = FeatureHighlightsManager.shared
        
        manager.markAsSeen(tab: .home)
        manager.markAsSeen(tab: .settings)
        XCTAssertFalse(manager.shouldShowHighlight(for: .home))
        
        manager.resetAll()
        
        XCTAssertTrue(manager.shouldShowHighlight(for: .home))
        XCTAssertTrue(manager.shouldShowHighlight(for: .settings))
    }

    // 测试场景 5：状态持久化验证
    func testStatePersistence_AfterReinitialization() {
        let manager = FeatureHighlightsManager.shared
        manager.markAsSeen(tab: .analytics)
        
        let sameInstance = FeatureHighlightsManager.shared
        
        XCTAssertFalse(sameInstance.shouldShowHighlight(for: .analytics))
    }
}
```

## 三、功能模块原则

### 3.1 模块划分
本次优化仅涉及 **单一模块**：

| 模块名称 | 职责 | 修改类型 |

### 9.3 关键技术决策记录

**决策1：为什么不用 @AppStorage？**
> @AppStorage 是 SwiftUI 属性包装器，只能在 struct View 中使用。
> FeatureHighlightsManager 是 class ObservableObject，无法直接使用 @AppStorage。
> 参考：Apple Documentation - @AppStorage 只适用于 View

**决策2：为什么选择 UserDefaults 而非其他方案？**
> - 项目已有 LocalProductRepository 使用 UserDefaults，保持一致性
> - 数据量小（仅5个Bool），无需 CoreData/SwiftData
> - 无需跨设备同步（iCloud同步是可选功能）
> - 实现简单，性能优秀

**决策3：为什么保留 @Published？**
> - @Published 提供 Combine 响应式能力
> - SwiftUI View 可以通过 @StateObject/@ObservedObject 监听变化
> - 未来如需实时UI更新，无需额外改造
